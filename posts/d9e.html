GitHub+Vercel通过部署hexo，绑定自己域名

注：建立在之前的失败安装上，借助Vercel提供的二级域名部署在网络中，省去了用GitHub二级域名发布的步骤。

优点：

  1. 步骤相对少点
  2. 可以直接创建私有库，这样自己有一些自定化的主题就不会公布出来.
  3. Vercel上可以把提供的二级域名换成自己的域名.

  • 准备：

    1. Vercel注册
    2. Github注册
    3. 安装node
    4. 安装git
    5. 安装vscode(可选，推荐)
    6. 域名(Vercel自带二级域名，国内连不上)

  • GitHub创建仓库

    跳转创建页面在Repositories中New一个仓库，Repository name输入hexo(都可以的),下面勾选Private然后直接Create repositories.

    之后进入hexo项目页面选择code再复制HTTPS里面的链接.

  • 命令行部分(windows推荐在git Bash Here中操作)

    注：先自行选择一个目录做为自己的blog文件夹

    1. 安装hexo-cli

    2. 初始化hexo，克隆github中作者提供的初始文件模板

    3. 克隆后会创建一个hexo文件夹，直接

    4. 安装其他所需组件

    5. 本地运行,默认端口4000，可指定

    6. 然后浏览器访问http://localhost:4000/，至此本地初始模板搭建运行完成

    7. 配置本地与github连接

      • 生成本地.ssh文件
      然后回车，接下来也不用填对应信息全部回车即可，最后会出现图下字符样式，之后用户(users)文件夹下的<你的用户名称文件夹内>会生成.ssh文件

    8. 复制.ssh文件夹下id_rsa.pub文件内的内容

    9. 在github的ssh中粘贴内容 点击直达 点击New SSH key，Title谁便填，Key type默认选Autentication Key，Key中粘贴id_rsa.pub中复制的内容.然后Add SSH key

    10. 之后返回Bash，输入

    11. 至此你可以在Bash中或者CMD中push本地文件到github了，push需要三步(具体用法Google，后面也可直接用hexo d命令上传GitHub就行)

    12. 使用hexo d命令上传，需在hexo提供了更快捷的提交到GitHub的命令，用vscode打开项目根文件hexo配置.config.yml文件，直接Ctrl+F搜索deploy关键字，在repository中添加GitHub中hexo项目的SSH引用，如：git@github.com:xxxxxxx/hexo.git (xxxxxxx为你的用户名)，branch分支默认main就行，完整代码如下：

    13. 保持刚刚的修改，在VScode自带的Terminal下输入如下命令

  • Verce增加一个项目

    1. Add New 一个prroject

    2. 在左边Import Git Repository选择Continue With GItHub，选择自己的hexo项目，Project Name自取，Framework Preset一定一定不要选hexo，默认Other就行，然后Deploy，部署成功后会提供一个二级域名就可以访问了，但前提需要外网才能访问.

    3. 绑定自己的域名，规避Vercel提供的免费二级域名被墙问题.

      进入Vercel部署的项目中，点Setitngs-Domains，框中输入自己的域名Add，然后根据提供的值，自己在域名接管商设置dns.

    4. 现在就能通过自己域名访问hexo
